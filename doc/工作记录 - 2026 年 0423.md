## 改动背景

任务详情右侧面板在窄宽度下，子任务区域的标题、截止时间、负责人和“详情”按钮会挤在一行里互相顶开，和你给的截图一致，表现出来就是布局发乱、信息不成块。

## 本轮补充记录：增大任务详情抽屉默认宽度

### 改动背景

你这次反馈的是任务详情右侧抽屉宽度偏窄，子任务区域里的标题、截止时间、负责人和“详情”按钮容易挤到两行显示。当前子任务行布局已经允许在窄宽度下换行兜底，但这次更合适的处理是把抽屉默认宽度加大，让默认打开时尽量保持单行展示。

### 主要修改点

### 1. 调大任务详情抽屉宽度常量

- 文件：`src/components/TaskDetailPanel/index.tsx`
- 处理内容：
  - `DETAIL_PANEL_DEFAULT_WIDTH` 从 `460` 调整到 `560`。
  - `DETAIL_PANEL_MIN_WIDTH` 从 `440` 调整到 `520`。
- 原因：
  - 默认抽屉宽度更宽后，子任务元信息区不用在刚打开时就换到第二行；最小宽度同步提高，避免用户拖窄后很快回到拥挤状态。

### 2. 同步详情抽屉基础样式

- 文件：`src/components/TaskDetailPanel/index.less`
- 处理内容：
  - `.detail-panel` 的 `width` 改成 `560px`。
  - `.detail-panel` 的 `min-width` 改成 `520px`。
- 原因：
  - 常量控制运行时宽度，Less 控制基础样式，两边需要保持一致，避免首屏样式和 React 状态宽度不一致。

### 3. 更新宽度回归测试

- 文件：
  - `tests/task-detail-rich-input-regression.test.mjs`
  - `tests/task-detail-drawer-regression.test.mjs`
- 处理内容：
  - 把默认宽度和最小宽度断言更新为 `560/520`。
  - 新增抽屉样式层的宽度断言。
- 原因：
  - 防止后续又把抽屉默认宽度改回偏窄，导致子任务区域再次默认双行。

### 异常处理

- 这次只调整抽屉宽度，不改子任务接口、子任务数据和负责人/截止时间交互。
- 子任务行原有的窄宽度换行兜底保留，避免极端拖窄或长标题时布局溢出。

### 可能影响范围

- 任务详情抽屉默认打开宽度
- 任务详情抽屉拖拽后的最小宽度
- 子任务列表默认展示密度

## 本轮补充记录：修复 Windows 关闭任务详情页滚动条抖动

### 改动背景

当前 Windows 浏览器关闭任务详情页时，会看到竖向滚动条宽度变化带来的抖动；Mac 浏览器默认多是覆盖式滚动条，滚动条不占内容宽度，所以不容易复现。

这次修复重点是稳定任务详情页内部滚动容器的滚动条占位，避免关闭详情页时内容区宽度因为滚动条出现或消失来回变化。

### 主要修改点

### 1. 给详情页滚动区预留稳定滚动条占位

- 文件：
  - `src/components/TaskDetailPanel/index.less`
- 处理内容：
  - 在 `.detail-scroll` 上新增 `scrollbar-gutter: stable;`。
- 原因：
  - Windows 传统竖向滚动条会占用宽度，如果滚动区在关闭时从有滚动条切到无滚动条，内容宽度会跳动。
  - 预留稳定 gutter 后，滚动条出现或消失时内容区宽度不再反复变化。

### 2. 补详情抽屉滚动条稳定性回归测试

- 文件：
  - `tests/task-detail-drawer-regression.test.mjs`
- 处理内容：
  - 新增断言，要求 `.detail-scroll` 必须保留 `scrollbar-gutter: stable;`。
- 原因：
  - 这个问题和平台滚动条机制有关，后续改详情页布局时容易误删，所以用测试固定下来。

### 异常处理

- 这次只动详情页内部滚动容器样式，不改关闭逻辑、不改任务数据、不改外层页面布局。
- 保留原来的 `overflow-y: auto`，不会影响详情页内容很多时的正常滚动。
