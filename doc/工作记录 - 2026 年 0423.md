任务详情右侧面板在窄宽度下，子任务区域的标题、截止时间、负责人和“详情”按钮会挤在一行里互相顶开，和你给的截图一致，表现出来就是布局发乱、信息不成块。

## 本轮补充记录：增大任务详情抽屉默认宽度

### 改动背景

### 主要修改点

### 1. 调大任务详情抽屉宽度常量

- 文件：`src/components/TaskDetailPanel/index.tsx`

- 原因：
  - 常量控制运行时宽度，Less 控制基础样式，两边需要保持一致，避免首屏样式和 React 状态宽度不一致。

### 3. 更新宽度回归测试

- 文件：
  - `tests/task-detail-rich-input-regression.test.mjs`

  - 新增抽屉样式层的宽度断言。
- 原因：
  - 防止后续又把抽屉默认宽度改回偏窄，导致子任务区域再次默认双行。

## 本轮补充记录：恢复任务表格列宽和铺满布局

### 改动背景

你这次反馈的是之前已经改好的任务表格布局被误回滚了，需要重新恢复。

这轮重点是把表格恢复成当前 Antd Table 上的可拖拽列宽，同时把除任务标题外的所有字段统一成适中的默认宽度；拖动某个字段时，只改变这个字段本身的宽度，不再带着其他字段一起挪位置；表格整体也不能缩成最小宽度显示。

### 主要修改点

### 1. 恢复表头列宽拖拽能力

- 文件：`src/components/TaskTable/index.tsx`
- 处理内容：
  - 恢复 `ResizableHeaderCell`。
  - 通过 Antd Table 的 `components.header.cell` 接入自定义表头单元格。
  - 给主要字段列和自定义字段列统一包一层 `withResizableHeader`。
- 原因：
  - 这样不替换表格组件，只在当前 Antd Table 上增加列宽拖拽能力，改动范围更小。

### 2. 非标题字段统一默认宽度

- 文件：`src/components/TaskTable/index.tsx`
- 处理内容：
  - 除任务标题外，普通字段默认宽度统一为 `120`。
  - 自定义字段默认宽度跟随普通字段，也统一为 `120`。
  - 任务标题列默认宽度和最小宽度统一为 `140`。
  - 删除普通字段单独的 `width: 96/120/156` 等历史宽度覆盖，避免初始状态宽度不一致。
- 原因：
  - 你的最新要求是除任务标题外，其余字段使用统一且大小适中的默认显示宽度，基本能显示“开始时间 / 截止时间”这类字段。
  - 如果字段名或字段值超过默认宽度，就用省略号显示，不再为了完整内容把默认列宽撑开。

### 3. 用占位列吸收剩余宽度

- 文件：`src/components/TaskTable/index.tsx`
- 处理内容：
  - 在快速新增字段列后追加 `tableLayoutSpacer` 占位列。
  - `tableScrollX` 只统计真实字段列宽，不统计占位列。
  - 表格继续使用 `tableLayout="fixed"`。
- 原因：
  - 真实字段宽度固定后，多出来的容器宽度交给占位列吸收。
  - 这样拖动某一列时，只会影响当前列宽，不会把其他真实字段一起重新分摊位置。

### 4. 恢复表格铺满和拖拽样式

  - 保持不再覆盖 Antd 内部 table 的 `min-width: 0`。
- 原因：
  - 不能再把整张表压成最小宽度显示，Antd 自身的横向滚动表格会保留 `min-width: 100%`，剩余宽度由占位列承接。
