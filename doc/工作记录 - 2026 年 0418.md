- 调整了 `vite.config.ts` 的 `manualChunks` 配置

- 把业务代码里的 `antd` 根入口导入改成 `antd/es/...` 的按组件导入

- 当前构建产物已经拆成多个 chunk：
  - `antd-base`
  - `antd-controls`
  - `antd-display`
  - `antd-feedback`
  - `antd-icons`
- 最大 chunk 已经低于 500 kB，构建告警消失

## 补充记录：潜在 bug 排查与修复

### 排查背景

这轮主要是对当前项目做一轮潜在 bug 排查，不只是看能不能编译，还重点看：

- 是否有明显的运行时风险
- 是否有数据展示错误
- 是否有容易触发重复提交、状态覆盖、竞态更新的问题
- 是否有 lint 已经能明确指出的 React 写法问题

### 修复内容

#### 1. 修复列表页数据加载竞态

- 原来 `TaskList.tsx` 里切换清单或视图时，旧请求有机会晚回来，把新视图数据覆盖掉
- 现在加了请求版本号保护，只允许最后一次请求落状态
- 同时把刷新动作改成递增版本触发，避免在 `effect` 里直接同步触发 `setState`

#### 2. 修复分组折叠状态被重置

- 原来 `TaskTable` 会在 `sections` 变化时直接重置 `collapsedSections`
- 用户手动展开或收起分组后，一刷新就会被覆盖
- 现在改成只在组件初始化时按默认值生成，切换不同清单时靠 `key` 重建组件

#### 3. 修复新建任务可能重复提交

- 原来输入框同时绑定了 `Enter` 和 `blur` 提交
- 回车后紧接着失焦，有机会重复创建同一个任务
- 现在增加了提交中的分组标记，重复事件不会二次落库

#### 4. 修复优先级文案显示错误

- 原来任务表格里优先级标签文案被写死成了“高”
- 现在按真实 `PriorityLabel` 显示，不同优先级能正确呈现

#### 5. 修复详情面板的清单与分组信息写死

- 原来任务详情里会固定显示“任务清单 1 / 默认分组”
- 现在会根据任务真实挂载的 `tasklist_guid` 和 `section_guid` 反查显示

#### 6. 修复详情面板创建人显示错误

- 原来评论区“创建了任务”显示的是当前登录人，不一定是实际创建人
- 现在改成按任务真实 `creator.id` 显示创建人名称

#### 7. 修复侧栏树节点的非空断言风险

- 原来 `Sidebar` 里有 `filter(Boolean)` 后继续 `tl!` 强断言的写法
- 数据一旦异常，运行时容易炸
- 现在改成显式类型守卫，去掉不安全断言

  - 月份面板再点一次可以切回日期面板
- 左右箭头会根据当前面板类型切换：
  - 日期面板按月翻

- 在月份面板里选中月份后，只切换当前浏览月份，再回到日期面板，不会误把“开始时间/截止时间”直接写掉

根因不是详情面板本身，而是页面最外层布局：

- 侧栏收起后，`Sider` 节点直接被移除了

- 给页面中间区域补了独立的 `app-main-layout`
- 把根布局 `app-layout` 固定成横向 `flex`
- 给中间主内容区补了 `min-width: 0`
- 给 `app-main-layout` 也补了 `min-width: 0`

这样侧栏收起以后，页面仍然保持三段式横向排布：

- 左侧边栏（有则显示）
- 中间任务列表内容
- 右侧详情面板

### 异常处理手法

- 这次没有继续改详情面板拖拽逻辑
- 修复只放在页面布局层，避免把详情内容区逻辑带乱
- 通过 `min-width: 0` 防止中间内容区在详情面板放大时把整体布局撑坏

## 补充记录：创建链路对标飞书并补齐 antd 交互

### 背景

这轮重点不是继续堆静态页面，而是把“创建分组 → 创建清单 → 创建任务分组 → 创建任务 → 创建子任务”这 5 条高频链路先做顺。因为当前项目还没接后端，后面真正联调时，页面交互如果先不对，越往后改越容易返工。

结合飞书任务页当前的实际形态，这轮优先对齐 3 件事：

- 每个阶段到底需要配哪些基础项
- 这些配置项是怎么和用户交互的
- 新增项出现时，页面的反馈和动效是不是足够自然

### 飞书链路拆解结论

#### 1. 创建分组

- 交互形态偏轻
- 更像行内新增，不打断当前页面上下文
- 核心配置项只有分组名称
- 用户动作主要是：
  - 点击“新建分组”
  - 原地出现输入框
  - 回车确认或失焦确认

#### 2. 创建清单

- 入口通常是左侧树区域里的加号菜单
- 先选“空白清单”或模板，再进入具体创建
- 相比创建分组，创建清单会多一点配置感
- 核心配置项当前先按静态页落了：
  - 清单名称
  - 清单颜色
  - 协作者
- 用户交互更像轻弹层，不是重 modal

#### 3. 创建任务分组

- 也是轻量行内交互
- 核心配置项只有分组名称
- 新建后会立刻插入当前清单列表中
- 页面反馈更重要，不能靠整页刷新

#### 4. 创建任务

- 飞书这条链路的真实感重点在“标题先行，配置补充”
- 所以这轮没有上重表单，而是做成表格内联创建
- 当前补的基础配置项有：
  - 标题
  - 负责人
  - 优先级
  - 开始时间
  - 截止时间
- 动画采用轻量插入和高亮闪现，避免新建后用户找不到刚加的任务

#### 5. 创建子任务

- 放在任务详情侧栏里原地完成
- 交互方式同样偏轻量，不离开当前任务上下文
- 当前补的基础配置项有：
  - 子任务标题
  - 负责人
  - 截止时间
- 创建后会回写父任务 `subtask_count`

### 本轮主要修改点

#### 1. Sidebar 补齐“创建清单 / 创建分组”链路

- `新建分组` 继续保持行内输入
- `创建清单` 改成：
  - 从加号下拉进入
  - 右侧轻弹层填写名称、颜色、协作者
  - 创建后本地增量插入树节点并自动选中
- 补了“新建中”占位状态和轻量淡入反馈

#### 2. TaskTable 补齐“创建任务分组 / 创建任务”链路

- `新建分组` 不再只改表格局部状态，同时同步回页面层 `tasklists`
- `新建任务` 改成任务表格内联创建卡片
- 创建时支持补充：
  - 负责人
  - 优先级
  - 开始时间
  - 截止时间
- 创建按钮改成显式确认，避免点时间选择器或下拉时误提交
- 新建任务和新建分组都补了插入动画

#### 3. TaskDetailPanel 补齐“创建子任务”链路

- `添加子任务` 先展开内联创建区
- 支持填写子任务标题、负责人、截止时间
- 创建后：
  - 子任务会本地追加到当前详情区列表
  - 父任务的 `subtask_count` 会同步更新
- 切换任务时，会重置评论、描述、子任务创建态，避免状态串台

#### 4. mock 数据层补齐创建链路支撑

- `CreateTaskPayload` 增加 `parent_task_guid`
- `createTask` 支持创建子任务并同步递增父任务计数

### 异常处理和风险控制

#### 1. 避免创建链路整页刷新

- 清单、分组、任务、子任务这几条链路现在都优先本地增量更新
- 这样能减少闪动，也能保住当前展开态、选中态和正在查看的详情态

#### 2. 避免任务创建误提交

- 现在改成显式“创建 / 取消”按钮，更稳

#### 3. 避免详情侧栏状态串台

- 原来如果切换任务，局部 state 可能残留到下一个任务
- 现在切换任务时会主动重置本地编辑态，并重新拉对应子任务
