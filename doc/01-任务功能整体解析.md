# 飞书"任务"功能深度解析

> 基于 lark-cli API schema 和实际数据调研
> 调研时间: 2026-04-17

---

## 一、核心概念

```
任务(Task) —— 最小工作单元，比如"完成需求评审"
清单(Tasklist) —— 任务的集合容器，比如"4月迭代"
分组(Section) —— 清单内的分类，比如"进行中""已完成""待开始"
子任务(Subtask) —— 任务的拆解，依附于父任务存在
成员(Member) —— 分"负责人"(assignee)和"关注人"(follower)
自定义字段(Custom Fields) —— 清单级别的扩展属性配置
```

---

## 二、任务数据模型

### 2.1 任务核心字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `guid` | string | 自动生成 | 任务全局唯一 ID（UUID 格式） |
| `task_id` | string | 自动生成 | 任务界面 ID（如 t107997） |
| `summary` | string | **是** | 任务标题，最大 3000 字符 |
| `description` | string | 否 | 任务备注/描述，最大 3000 字符 |
| `status` | string | 自动 | `todo` 或 `done` |
| `completed_at` | string | 自动 | 完成时间戳(ms)，0 或空表示未完成 |
| `created_at` | string | 自动 | 创建时间戳(ms) |
| `updated_at` | string | 自动 | 最后更新时间戳(ms) |
| `creator` | Member | 自动 | 创建者信息 |

### 2.2 任务层级

**最多 2 级：父任务 + 子任务，不支持更深层级**

- 每个任务通过 `parent_task_guid` 字段关联父任务
- 子任务通过 `subtasks list` API 获取
- 父任务有 `subtask_count` 字段记录子任务数量

## 三、任务来源说明（MVP 聚焦）

API 支持 6 种来源，但 MVP 阶段**只实现任务中心创建**（source=1）。

| source 值 | 来源 | MVP阶段 |
|-----------|------|---------|
| **1** | **任务中心** | **✅ 实现** |
| 0 | 未知来源 | ❌ |
| 2 | 群组任务 / 消息转任务 | ❌ 后续扩展 |
| 6 | 开放平台 tenant_access_token 创建 | ❌ 后续扩展 |
| 7 | 开放平台 user_access_token 创建 | ❌ 后续扩展 |
| 8 | 文档任务（从文档中创建） | ❌ 后续扩展 |

## 四、创建任务时可配置的完整内容

### 4.1 基本信息
- `summary` — 任务标题（必填）
- `description` — 任务描述
- `is_milestone` — 是否为里程碑任务

### 4.2 时间配置
- `start` — 开始时间（timestamp + is_all_day）
- `due` — 截止时间（timestamp + is_all_day）
- `repeat_rule` — 重复规则（iCalendar RRULE 格式）

### 4.3 成员配置（members）
- `role`: `assignee`（负责人）或 `follower`（关注人）
- `type`: `user` 或 `app`
- 最多 50 个成员（去重后）

### 4.4 完成模式（mode）
- `1` — 会签任务：所有负责人都需完成
- `2` — 或签任务：任一负责人完成即可

### 4.5 提醒（reminders）
- `relative_fire_minute` — 相对于截止时间的提前提醒分钟数
- 最多只能设置 **1 个提醒**
- 必须同时设置截止时间

### 4.6 清单归属（tasklists）
- `tasklist_guid` — 所属清单 GUID
- `section_guid` — 清单内的分组 GUID

### 4.7 自定义字段（custom_fields）
支持 6 种类型：

| 类型 | 字段 | 说明 |
|------|------|------|
| number | `number_value` | 数字 |
| member | `member_value` | 人员（数组） |
| datetime | `datetime_value` | 日期时间戳 |
| single_select | `single_select_value` | 单选选项 GUID |
| multi_select | `multi_select_value` | 多选选项 GUID 数组 |
| text | `text_value` | 文本（3000字符） |

### 4.8 优先级（MVP 一等字段）

实际产品中优先级是任务的一等字段（非自定义字段），前端以图标+颜色区分：

| 值 | 含义 | 图标颜色 |
|------|------|----------|
| 无 | 未设置 | 灰色 |
| 低 | 低优先级 | 蓝色 |
| 中 | 中优先级 | 黄色 |
| 高 | 高优先级 | 橙色 |
| 紧急 | 紧急 | 红色 |

> 注：飞书 API 中优先级可能通过 `custom_fields` 或专用字段传递，前端需将其作为独立字段展示在任务卡片和详情页中。

### 4.9 附件（attachments）

实际产品支持在任务详情中添加附件：
- 详情面板有"添加附件"入口
- 支持上传文件（图片、文档等）
- 附件列表在详情页中独立展示

> 注：MVP 阶段实现附件展示和上传的基础 UI，具体 API 以实际对接为准。

### 4.10 任务依赖（dependencies）
- `prev` — 前置依赖
- `next` — 后置依赖

### 4.11 其他
- `extra` — 自定义附带数据（Base64 编码）
- `client_token` — 幂等 token
- `positive_reminders` — 正数协议每日提醒

## 五、任务状态机

```
[新建] ──── todo 状态
   │
   ├─→ 勾选完成 ──── done 状态（记录 completed_at）
   │                    │
   │                    └─→ 重新打开 ──── todo 状态
   │
   └─→ 删除 ──── 任务消失
```

## 六、用户操作视图

### 6.1 左侧导航结构

实际产品左侧导航比文档最初描述更丰富：

```
左侧导航
├── 我负责的        ← 对应"我的任务"
├── 我关注的        ← 对应"我相关的"
├── 动态            ← 任务变更动态流（MVP 做占位页）
├── 来自飞书项目     ← 项目关联任务（MVP 做占位页）
├── 快速访问        ← 收藏/置顶的清单快捷入口（MVP 做占位页）
├── ── 分割线 ──
├── 清单分组A（如"测试分组"）  ← 清单本身可被分组管理
│   ├── 清单1
│   └── 清单2
├── 清单分组B
│   └── 清单3
└── 未分组清单
    └── 清单4
```

> 注：MVP 实现"我负责的""我关注的""清单"三个核心入口，其余导航项做占位/假页面。

### 6.2 查看维度
- **我负责的** —— 我是负责人的任务
- **我关注的** —— 我是关注人的任务
- **清单视图** —— 按清单+分组组织查看
- **搜索结果** —— 按关键词/负责人/创建者/完成状态/时间范围搜索

### 6.3 筛选条件
- 按负责人（assignee）
- 按关注人（follower）
- 按创建者（creator）
- 按完成状态（completed / incomplete）
- 按截止时间范围（due）
- 按关键词（query）

### 6.4 排序与分组

实际产品任务列表支持多种排序和分组方式：

**排序方式：**
- 按创建时间（默认）
- 按截止时间
- 按优先级
- 拖拽自定义排序

**分组方式：**
- 按清单分组（默认）
- 按优先级分组
- 按截止时间分组
- 自定义分组

> 注：MVP 仅实现按创建时间排序 + 按清单分组展示，其他排序和分组方式后续迭代。

### 6.5 字段配置（列表视图）

实际产品列表视图的工具栏有"字段配置"功能：
- 用户可选择在列表中显示/隐藏哪些列
- 可配置的列包括：负责人、截止时间、优先级、创建时间、自定义字段等
- 列顺序可拖拽调整

> 注：MVP 提供固定列展示（标题、负责人、截止时间、优先级），字段配置功能后续迭代。

## 七、清单(Tasklist)体系

### 7.1 清单数据模型

| 字段 | 说明 |
|------|------|
| `guid` | 清单全局唯一 ID |
| `name` | 清单名，最长 100 字符 |
| `owner` | 清单所有者 |
| `creator` | 清单创建者 |
| `members` | 协作成员列表 |
| `archive_msec` | 归档时间戳（0 表示未归档） |
| `created_at` | 创建时间戳 |
| `updated_at` | 最后更新时间戳 |
| `url` | 清单分享链接 |

### 7.2 成员角色权限

| 权限 | owner | editor | viewer |
|------|-------|--------|--------|
| 编辑清单名称 | ✅ | ✅ | ❌ |
| 管理成员 | ✅ | ❌ | ❌ |
| 转移所有权 | ✅ | ❌ | ❌ |
| 在清单中创建任务 | ✅ | ✅ | ❌ |
| 编辑清单内任务 | ✅ | ✅ | ❌ |
| 查看清单任务 | ✅ | ✅ | ✅ |
| 删除清单 | ✅ | ❌ | ❌ |

## 八、关键限制

1. **任务层级**：最多 2 级（父任务 + 子任务）
2. **标题/描述**：最大 3000 UTF-8 字符
3. **成员数**：单次操作最多 50 个
4. **提醒数**：每个任务最多 1 个
5. **自定义字段**：必须关联到任务所在清单
6. **来源信息**：创建后不可更改（origin）
7. **清单名称**：最长 100 字符
8. **任务跨清单**：实际产品中 `tasklists` 是数组，一个任务可属于多个清单。**MVP 暂不实现跨清单，一个任务只归属一个清单**

## 九、API 操作能力

| 操作 | API 路径 | 所需权限 |
|------|----------|----------|
| 获取我的任务 | GET `/open-apis/task/v2/tasks` | task:task:read |
| 获取任务详情 | GET `/open-apis/task/v2/tasks/:guid` | task:task:read |
| 创建任务 | POST `/open-apis/task/v2/tasks` | task:task:write |
| 更新任务 | PATCH `/open-apis/task/v2/tasks/:guid` | task:task:write |
| 删除任务 | DELETE `/open-apis/task/v2/tasks/:guid` | task:task:write |
| 子任务列表 | GET `/open-apis/task/v2/tasks/:guid/subtasks` | task:task:read |
| 创建子任务 | POST `/open-apis/task/v2/tasks/:guid/subtasks` | task:task:write |
| 添加评论 | POST `/open-apis/task/v2/tasks/:guid/comments` | task:task:write |
| 搜索任务 | POST `/open-apis/task/v2/tasks/search` | task:task:read |
| 清单列表 | GET `/open-apis/task/v2/tasklists` | task:tasklist:read |
| 创建清单 | POST `/open-apis/task/v2/tasklists` | task:tasklist:write |
| 更新清单 | PATCH `/open-apis/task/v2/tasklists/:guid` | task:tasklist:write |
| 清单任务列表 | GET `/open-apis/task/v2/tasklists/:guid/tasks` | task:tasklist:read |
