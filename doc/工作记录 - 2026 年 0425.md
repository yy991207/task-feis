# 工作记录 - 2026 年 0425

## 改动背景

根据接口调整说明，前端需要从原来的 team 维度分组接口切换到个人分组接口。核心变化是分组接口路径从 `project-groups` 切换到 `user-project-groups`，清单归属字段从 `group_id` 增加到 `user_group_id`，清单筛选参数也改为 `user_group_id`。

## 主要修改点

1. 分组接口服务调整
   - `GET /teams/{team_id}/project-groups` 改为 `GET /user-project-groups?user_id=xxx`。
   - `POST /teams/{team_id}/project-groups` 改为 `POST /user-project-groups`。
   - `PUT /project-groups/{group_id}` 改为 `PUT /user-project-groups/{group_id}`。
   - `DELETE /project-groups/{group_id}` 改为 `DELETE /user-project-groups/{group_id}`。
   - `PATCH /project-groups/{group_id}/sort-order` 改为 `PATCH /user-project-groups/{group_id}/sort-order`。

2. 清单接口服务调整
   - `listProjects` 的分组筛选参数从 `group_id` 改为 `user_group_id`。
   - 拖拽清单到分组的接口从 `PATCH /projects/{project_id}/group` 改为 `PATCH /projects/{project_id}/user-group`。
   - 拖拽 body 保持提交 `user_id` 和 `group_id`，与新接口说明一致。

3. 类型字段补齐
   - `ProjectGroup` 新增 `owner_user_id` 和 `project_count`。
   - `Project` 新增 `user_group_id` 和 `user_group_name`。

4. 侧栏分组数据适配
   - 新增 `getProjectGroupId`，优先读取 `user_group_id`，旧数据回退到 `group_id`。
   - 侧栏项目分组、未分组判断、拖拽同组判断和本地乐观更新都统一走这个方法。

## 异常处理手法

- 原有请求封装仍统一处理 HTTP 错误和业务错误码，这次没有改动错误处理入口。
- 拖拽分组失败时继续保留原有回滚逻辑：接口失败后恢复 `prevProjects`，并弹出错误提示。
- 真实接口测试脚本使用 `finally` 清理临时创建的分组，避免测试数据残留。

## 补充记录：任务编辑态改为 Ant Design 组件渲染

### 改动背景

这轮需求很明确：任务表里“新建任务”和“已有任务进入编辑态”这两种场景，之前那套输入框和整行编辑效果主要靠自定义 `div + less` 壳子撑出来。现在要把这块收回到 Ant Design 组件上，避免继续维护一套看起来像控件、其实不是控件的自定义结构。

### 根因

- 当前表格主体本来就是 Ant Design `Table`，但编辑态这一层又额外包了很多自定义结构，比如：
  - `inline-create-title-cell`
  - `inline-create-field-shell`
  - `task-title-edit-box`
- 这些结构把样式、交互和布局绑得太死，后面只要调整一处输入框或按钮外观，就要同时改 JSX 和一大段 less。
- 真正的问题不是业务逻辑，而是编辑态渲染层没有直接复用 antd 自己的 `Input`、`Button`、`Select` 这些组件能力。

### 主要修改点

1. 标题编辑框直接改成 antd `Input`
   - 文件：`src/components/TaskTable/index.tsx`
   - `TaskTitleEditBox` 不再外包自定义 `task-title-edit-box` 容器。
   - 直接把激活态样式挂到 antd `Input` 本身。
   - 行内新建任务标题用 `prefix` 把禁用态 `Checkbox` 收进输入框，替掉原来左边单独的自定义格子。

2. 行内新建任务这一排改成 antd 组件组合
   - 文件：`src/components/TaskTable/index.tsx`
   - 标题区域改用 antd `Flex`。
   - 优先级直接使用 antd `Select`。
   - 负责人触发器改成 antd `Button`。
   - 日期触发器改成 antd `Button`，保留原来的 Popover 和日期面板逻辑。
   - 创建人展示区继续用 `Space + Avatar`，但外层不再套旧的自定义字段壳。

3. 已有任务编辑态同步收口
   - 文件：`src/components/TaskTable/index.tsx`
   - 已有任务标题重命名复用同一个 `TaskTitleEditBox`。
