# 改动背景

- 侧栏里的“任务清单”根节点在空目录时，没有像“新分组”那样显示展开/收起三角按钮。
- 同一个空目录场景下，根节点下面也缺少“可拖拽清单加入该分组”的提示词，导致交互和分组区不一致。

# 根因分析

- 左侧树组件里，“新分组”空态会插入一个占位子节点，所以 `Tree` 会把它当成可展开节点处理。
- “任务清单”根节点在空态时直接传了空数组，`Tree` 会把它当成叶子节点，所以三角按钮和占位提示词都会消失。
- 另外排查时发现，侧栏有几条源码正则回归测试已经落后于当前实现，测试目标没问题，但断言写法和现状不一致，导致会误报。

# 主要修改点

- 在 `src/components/Sidebar/index.tsx` 里补了根节点空态分支：
  - 当 `ungroupedProjects` 为空且默认分组存在时，复用现有的空分组占位节点构造逻辑。
  - 这样根节点空态也会稳定显示展开按钮和“可拖拽清单加入该分组”提示词。
- 在 `src/components/Sidebar/index.less` 里把分组行隐藏拖拽占位重新收窄到 `9px`，和现有中文注释保持一致，避免展开箭头列错位。
- 新增回归测试 `tests/sidebar-root-empty-placeholder-regression.test.mjs`，专门覆盖根节点空态占位逻辑。
- 顺手把几条已有侧栏源码回归测试更新到当前实现语义，避免旧断言误报：
  - `tests/sidebar-empty-group-switcher-regression.test.mjs`
  - `tests/default-tasklist-overview-regression.test.mjs`
  - `tests/sidebar-tree-switcher-alignment-regression.test.mjs`

# 异常处理手法

- 这次没有新增接口调用，也没有改接口异常处理链路。
- 主要还是复用现有空分组占位节点的拖拽事件处理，保持根节点空态和普通分组空态走同一套逻辑，减少分叉。
