# MVP 范围：任务中心创建任务的完整流程

> 聚焦任务中心（source=1）作为唯一入口，后续再扩展其他数据源
> 2026-04-17

---

## 一、MVP 范围界定

### 做什么 ✅
- 从**任务中心**创建任务
- 创建/管理清单和分组
- 在清单中创建任务
- 任务的查看/编辑/完成/删除
- 任务成员管理（负责人/关注人）
- 子任务
- 截止时间 + 提醒
- 任务评论
- 任务依赖
- 自定义字段（清单级别）

### MVP 不做 ❌
- 消息转任务（source=2）
- 文档任务（source=8）
- API/机器人创建任务（source=6/7）
- 把已有任务移入清单
- 重复任务（repeat_rule）
- 任务跨清单
- 正数协议每日提醒（positive_reminders）

## 二、任务中心创建任务 — 完整操作链路

### 第一步：进入任务中心

```
用户点击飞书侧边栏"任务"
    ↓
进入任务中心首页
    ├── "我的任务" tab（默认）
    ├── "我相关的" tab
    └── "清单" tab
```

### 第二步：创建任务（两条路径）

#### 路径 A：游离任务（不归属任何清单）

```
点击"新建任务"按钮
    ↓
弹出任务创建表单
    ├── 填写标题（必填，最长3000字符）
    ├── 填写描述（可选）
    ├── 设置负责人（可选，支持多人）
    ├── 设置关注人（可选）
    ├── 设置截止时间（可选）
    ├── 设置开始时间（可选）
    ├── 设置提醒（可选，需先有截止时间）
    ├── 设置完成模式：或签/会签（可选，默认或签）
    ├── 设为里程碑（可选）
    ├── 设置优先级（可选，无/低/中/高/紧急）
    ├── 添加子任务（可选）
    └── 设置自定义字段（可选，仅当指定了清单时才显示）
    ↓
点击"创建"
    ↓
POST /open-apis/task/v2/tasks
Body: {
  "summary": "任务标题",           // 必填
  "description": "任务描述",       // 可选
  "members": [                     // 可选
    { "id": "ou_xxx", "role": "assignee" },
    { "id": "ou_yyy", "role": "follower" }
  ],
  "due": {                          // 可选
    "timestamp": "1714000000000",
    "is_all_day": false
  },
  "start": {                        // 可选
    "timestamp": "1713900000000",
    "is_all_day": false
  },
  "mode": 2,                        // 1=会签, 2=或签, 默认2
  "is_milestone": false,            // 可选
  "reminders": [                    // 可选
    { "relative_fire_minute": 30 }
  ]
}
    ↓
任务创建成功，出现在"我的任务"列表中
```

#### 路径 B：指定清单创建

```
进入"清单" tab → 选择目标清单 → 点击"新建任务"
    ↓
弹出任务创建表单（多一个字段）
    ├── 填写标题（必填）
    ├── 填写描述（可选）
    ├── 设置负责人（可选）
    ├── 设置关注人（可选）
    ├── 设置截止时间（可选）
    ├── 设置开始时间（可选）
    ├── 设置提醒（可选，需先有截止时间）
    ├── 设置完成模式（可选）
    ├── 设为里程碑（可选）
    ├── 选择分组（必填，清单内的分组） ← 与路径A的区别
    └── 设置自定义字段（可选，清单已配置的字段）
    ↓
POST /open-apis/task/v2/tasks
Body: {
  "summary": "任务标题",
  "tasklists": [{                    // ← 路径B特有
    "tasklist_guid": "清单GUID",
    "section_guid": "分组GUID"
  }],
  "members": [...],
  "custom_fields": [                 // ← 路径B可选
    {
      "guid": "字段GUID",
      "text_value": "高优先级"
    }
  ],
  ...其他字段同路径A
}
    ↓
任务创建成功，出现在指定清单的指定分组中
```

## 三、任务创建表单 — 字段完整定义

### 3.1 必填字段

| 字段 | 类型 | 限制 | 说明 |
|------|------|------|------|
| `summary` | string | 最大3000字符 | 任务标题，不可为空 |

### 3.2 可选字段

| 字段 | 类型 | 条件 | 说明 |
|------|------|------|------|
| `description` | string | 无 | 任务描述，最大3000字符 |
| `members` | array | 无 | 成员列表，最多50个 |
| `members[].role` | string | — | `assignee` 或 `follower` |
| `members[].id` | string | — | 用户 open_id |
| `due.timestamp` | string | — | 截止时间戳(ms) |
| `due.is_all_day` | boolean | — | 是否全天 |
| `start.timestamp` | string | — | 开始时间戳(ms) |
| `start.is_all_day` | boolean | — | 是否全天 |
| `reminders` | array | 需有due | 最多1个 |
| `reminders[].relative_fire_minute` | int | ≥0 | 截止前N分钟提醒 |
| `mode` | int | — | 1=会签, 2=或签(默认) |
| `is_milestone` | boolean | — | 是否里程碑 |
| `priority` | int | — | 优先级：0=无, 1=低, 2=中, 3=高, 4=紧急 |
| `tasklists` | array | — | 归属清单+分组 |
| `custom_fields` | array | 需在清单中 | 自定义字段值 |

### 3.3 字段联动规则

| 规则 | 说明 |
|------|------|
| 提醒 ↔ 截止时间 | 没有截止时间就不能设置提醒 |
| 自定义字段 ↔ 清单 | 未指定清单时不显示自定义字段 |
| 开始时间 ≤ 截止时间 | 同时设置时，开始时间必须 <= 截止时间 |
| 开始/截止 is_all_day 一致 | 同时设置时，两者的 is_all_day 必须相同 |
| 成员去重 | 同一用户不能同时出现在 assignee 和 follower 中 |

## 四、创建后状态

### 4.1 游离任务

```
创建后出现在：
├── "我的任务"（如果当前用户是负责人）
├── "我相关的"（如果当前用户是关注人）
└── 不属于任何清单
```

### 4.2 清单内任务

```
创建后出现在：
├── "我的任务"（如果当前用户是负责人）
├── 清单视图 → 指定分组中
└── 清单成员都能看到
```

### 4.3 通知行为

| 操作 | 通知对象 | 通知方式 |
|------|----------|----------|
| 创建任务指定负责人 | 所有 assignee | 飞书消息通知 |
| 创建任务指定关注人 | 所有 follower | 飞书消息通知 |
| 任务出现在对方"我的任务"或"我相关的" | — | — |

## 五、创建任务后的操作

### 5.1 编辑任务

```
点击任务卡片 → 进入任务详情
    ├── 编辑标题/描述
    ├── 修改截止时间/开始时间
    ├── 添加/移除成员
    ├── 切换会签/或签
    ├── 设置/取消里程碑
    ├── 设置/取消提醒
    ├── 添加/删除子任务
    ├── 添加评论
    └── 设置自定义字段（清单内任务）
```

**API：**
```
PATCH /open-apis/task/v2/tasks/{task_guid}
Body: {
  "task": { ...只传要改的字段... },
  "update_fields": ["summary", "due", "members", ...]
}
```

**可更新字段列表：**
- `summary` — 标题
- `description` — 描述
- `due` — 截止时间
- `start` — 开始时间
- `members` — 成员
- `mode` — 会签/或签
- `is_milestone` — 里程碑
- `reminders` — 提醒
- `custom_fields` — 自定义字段
- `tasklists` — 清单归属
- `dependencies` — 依赖关系

### 5.2 完成任务

```
点击任务前的 checkbox
    ↓
POST /open-apis/task/v2/tasks/{task_guid}/complete
    ↓
任务状态变为 done
记录 completed_at 时间戳
```

**会签任务特殊逻辑：**
```
mode=1（会签）时：
├── 每个 assignee 独立标记完成
├── 所有人完成前，任务状态仍是 todo
└── 所有人完成后，任务状态变为 done

mode=2（或签）时：
├── 任一 assignee 完成即可
└── 任务状态立即变为 done
```

### 5.3 重新打开

```
已完成的任务 → 点击"重新打开"
    ↓
POST /open-apis/task/v2/tasks/{task_guid}/reopen
    ↓
任务状态变回 todo
completed_at 清空
```

### 5.4 删除任务

```
任务详情页 → 更多操作 → 删除
    ↓
DELETE /open-apis/task/v2/tasks/{task_guid}
    ↓
任务永久删除
子任务一并删除
```

## 六、前端需要处理的边界情况

| 场景 | 处理方式 |
|------|----------|
| 标题为空 | 禁止提交，提示"标题不能为空" |
| 标题超长 | 限制3000字符，显示字数统计 |
| 截止时间早于开始时间 | 提示"截止时间不能早于开始时间" |
| 无截止时间设提醒 | 禁用提醒控件，提示"请先设置截止时间" |
| 自定义字段无清单 | 不显示自定义字段区域 |
| 会签任务完成 | 显示每个人的完成状态和进度 |
| 无任务时 | 显示空状态引导（"还没有任务，点击新建"） |
| 无清单时 | 清单tab显示空状态引导创建清单 |
| 权限不足 | 非负责人/创建者隐藏编辑按钮 |
| 网络失败 | 显示错误提示，保留表单内容 |
