# 工作记录 - 2026 年 0424

## 改动背景

这轮按“自定义字段选项 `value → id` 改造”文档，对前端做接口适配和展示链路收口。重点不是只改创建/更新字段的入参，还要把任务列表、动态、详情历史这些显示层一起切到新返回结构，避免出现“保存好了但展示还是旧字段”的半修状态。

## 主要修改点

### 1. 自定义字段服务层改成 `option.id`

- 文件：`src/services/customFieldService.ts`
- 调整点：
  - `FieldOption` 去掉旧的 `value` 字段，只保留 `id / label / color`。
  - 创建字段接口继续按新文档传 `options: [{ label, color }]`。
  - 更新字段接口继续按新文档传 `options: [{ id, label, color }]`。

### 2. 任务模型补了展示态字段

- 文件：
  - `src/services/taskService.ts`
  - `src/types/task.ts`
  - `src/services/projectService.ts`
- 调整点：
  - `ApiTask` 新增 `custom_fields_display`、`priority_option`。
  - `apiTaskToTask` 会把这两个字段一起映射到前端任务对象。
  - `Task` 类型新增可选的 `custom_fields_display`、`priority_option`，这样显示层可以直接消费后端给的 label/color。
  - 项目摘要服务层类型改成数组项结构，和新文档保持一致。

### 3. 任务表格切到新选项字段，并区分“写值”和“显示值”

- 文件：`src/components/TaskTable/index.tsx`
- 调整点：
  - 自定义字段定义映射从 `option.id` 生成前端 `guid`。
  - 字段筛选下拉从旧的 `option.value` 改成 `option.id`。
  - `formatCustomFieldValue` 对 `single_select / multi_select` 优先使用 `task.custom_fields_display` 显示 label。
  - 乐观更新自定义字段值时，同时补本地 `custom_fields_display`，避免保存后短时间显示原始 ID。
  - 优先级乐观更新时同步补 `priority_option`，避免前后展示不一致。

### 4. 字段编辑弹窗不再混用旧的 `value`

- 文件：`src/components/CustomFieldEditorModal/index.tsx`
- 调整点：
  - 新增本地 `DraftOption`，只用于弹窗里临时编辑行的 key。
  - 不再把服务层的 `FieldOption` 直接当成本地草稿结构用，避免把旧 `value` 写法带回接口层。

### 5. 动态和详情历史接住 `*_value_label`

- 文件：
  - `src/components/ActivityView/index.tsx`
  - `src/components/TaskDetailPanel/index.tsx`
- 调整点：
  - 新增活动值展示函数，优先显示 `new_value_label / old_value_label`，没有再回退到原始值。
  - 增加 `task.priority_changed` 的文案显示。
  - `task.custom_field_changed` 和 `task.status_changed` 也统一走新展示值。

## 异常处理手法

- 自定义字段值本地更新时先做乐观更新，接口失败再回滚，避免用户点完后表格没有即时反馈。
- 展示态字段统一做兜底：
  - 有 `custom_fields_display / priority_option` 就优先显示；
  - 没有就回退到原始值和本地选项表，避免接口灰度期间页面直接空掉。
