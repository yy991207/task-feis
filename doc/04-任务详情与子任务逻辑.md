## 一、任务详情页 — 完整结构

### 1.1 详情页展示的所有信息

```
任务详情页
├── 头部
│   ├── [ ] checkbox（完成/未完成）
│   ├── 标题（可编辑）
│   └── 更多操作按钮（...）
│       ├── 删除任务
│       └── 重新打开（仅已完成时显示）
│
├── 基本信息区
│   ├── 描述（可编辑）
│   ├── 负责人列表（assignee）
│   │   ├── 添加负责人按钮
│   │   └── 移除按钮（每个负责人旁）
│   ├── 关注人列表（follower）
│   │   ├── 添加关注人按钮
│   │   └── 移除按钮
│   ├── 截止时间（可编辑）
│   ├── 开始时间（可编辑）
│   ├── 完成模式标签（会签/或签，可切换）
│   ├── 优先级（无/低/中/高/紧急，图标+颜色区分）
│   └── 里程碑标记（可切换）
│
├── 提醒设置区

│
├── 子任务区

│   │   └── ...
│   ├── 子任务计数（共X个子任务，Y个已完成）
│   └── 添加子任务按钮
│
├── 附件区

├── 自定义字段区（仅清单内任务）
│   ├── 字段1: 值（可编辑）

│   └── ...
│
├── 任务依赖区
│   ├── 前置依赖：[任务A名称]（可添加/移除）
│   └── 后置依赖：[任务B名称]（可添加/移除）

└── 评论区
    ├── 评论列表
    │   ├── 用户A: 评论内容 + 时间
    │   └── 用户B: 评论内容 + 时间
    └── 输入框 + 发送按钮
```

## 二、任务详情 API

### 2.1 获取任务详情

```
GET /open-apis/task/v2/tasks/{task_guid}
```

**返回完整结构：**
```json

  "data": {
    "task": {
      "guid": "cd2a7fea-...",
      "task_id": "t107997",
      "summary": "智能体\"公开\"功能",
      "description": "",
      "status": "todo",
      "completed_at": "0",
      "created_at": "1776062619652",
      "updated_at": "1776062642651",
      "creator": {
        "id": "ou_3efae302...",
        "type": "user"
      },
      "mode": 2,
      "is_milestone": false,
      "source": 1,
      "parent_task_guid": "",
      "subtask_count": 0,
      "members": [
        { "id": "ou_e654b427...", "role": "assignee", "type": "user" },
        { "id": "ou_90894afa...", "role": "assignee", "type": "user" },

      "url": "https://applink.feishu.cn/client/todo/detail?guid=..."
    }

```

### 2.2 获取子任务列表

```
GET /open-apis/task/v2/tasks/{task_guid}/subtasks
```

**返回：**
```json
{
  "code": 0,
  "data": {
    "has_more": false,
    "items": [
      {
        "guid": "子任务GUID",
        "summary": "子任务标题",
        "status": "todo",
        "created_at": "1776000000000"
      }
    ],
    "page_token": ""
  }
}
```

### 2.3 创建子任务

```
POST /open-apis/task/v2/tasks/{parent_task_guid}/subtasks
Body: {
  "summary": "子任务标题",    // 必填
  "description": "子任务描述",
  "members": [
    { "id": "ou_xxx", "role": "assignee" }
  ],
  "due": { ... },
  "reminders": [ ... ]
}
```

**子任务的特点：**
- 子任务拥有父任务的所有可编辑属性
- 子任务的 `parent_task_guid` 自动设为父任务 GUID
- 子任务可以有自己的负责人（不必和父任务相同）
- 子任务完成后不影响父任务状态（无自动联动）
- 子任务不能再有子任务（最多2级）
- 删除父任务时，子任务一并删除

## 三、子任务的操作逻辑

### 3.1 添加子任务

```
任务详情页 → 子任务区 → 点击"添加子任务"
    ↓
输入子任务标题（回车创建）
    ↓
POST /open-apis/task/v2/tasks/{parent_guid}/subtasks
Body: { "summary": "子任务标题" }
    ↓
子任务创建成功
子任务计数 +1
出现在子任务列表中
```

### 3.2 完成子任务

```
点击子任务前的 checkbox
    ↓
子任务状态变为 done
父任务的 subtask_count 不变，但前端可计算已完成数量
```

### 3.3 从子任务进入子任务详情

```
点击子任务标题
    ↓
弹出子任务详情（与父任务详情结构相同）
    ├── 可编辑标题/描述
    ├── 可设置负责人/截止时间
    ├── 可添加评论
    └── 可设置自定义字段（如果父任务在清单中）
```

**关键区别：**
- 子任务详情页显示"所属父任务"信息
- 子任务详情页不可再添加子任务（2级限制）

## 四、任务评论逻辑

### 4.1 添加评论

```
任务详情页 → 评论区 → 输入内容 → 发送
    ↓
POST /open-apis/task/v2/tasks/{task_guid}/comments
Body: { "content": "评论内容" }
    ↓
评论出现在列表底部
通知其他任务成员有新评论
```

### 4.2 评论能力

实际产品中评论支持富文本，包括：
- 文字内容
- 插入图片（粘贴或上传）
- @提及人员

API 层面：
- 只能添加，API 未提供编辑/删除接口
- 评论与任务状态独立
- 所有任务成员都能看到评论

> 注：之前文档标注"纯文本，不支持富文本/图片/附件"有误，已修正。MVP 阶段实现文字+图片评论。

## 五、任务依赖逻辑（MVP 简化版）

### 5.1 设置依赖

```
任务详情页 → 依赖区 → 添加依赖
    ↓
选择前置任务或后置任务
    ↓
PATCH /open-apis/task/v2/tasks/{task_guid}
Body: {
  "task": {
    "dependencies": [
      { "task_guid": "xxx", "type": "prev" }
    ]
  },
  "update_fields": ["dependencies"]
}
```

### 5.2 依赖类型

| type | 含义 | 语义 |
|------|------|------|
| `prev` | 前置依赖 | 本任务依赖于那个任务 |
| `next` | 后置依赖 | 那个任务依赖于本任务 |

### 5.3 依赖的行为（MVP 阶段）

- **纯展示**：不自动阻止任务完成
- 前端展示依赖链（"等待任务A完成"）
- 被依赖任务完成后，前端可提示"前置任务已完成"

## 六、自定义字段操作逻辑

### 6.1 自定义字段只在清单内任务中可用

```
任务在清单中 → 详情页显示自定义字段区
    ├── 字段值可编辑
    └── 保存后更新到任务

任务不在清单中 → 不显示自定义字段区
```

### 6.2 编辑自定义字段

```
PATCH /open-apis/task/v2/tasks/{task_guid}
Body: {
  "task": {
    "custom_fields": [
      {
        "guid": "字段GUID",
        "text_value": "新值"
      }
    ]
  },
  "update_fields": ["custom_fields"]
}
```

### 6.3 字段类型与控件映射

| 字段类型 | 前端控件 | 值字段 |
|----------|----------|--------|
| text | 多行文本框 | `text_value` |
| number | 数字输入框 | `number_value` |
| datetime | 日期时间选择器 | `datetime_value` |
| single_select | 单选下拉框 | `single_select_value` |
| multi_select | 多选下拉框/标签选择器 | `multi_select_value` |
| member | 人员选择器（可多选） | `member_value` |

## 七、权限控制（MVP 简化版）

### 7.1 谁能编辑任务

| 角色 | 可编辑 | 可完成 | 可删除 | 可评论 |

## 八、会签任务的详情展示

### 8.1 详情中的完成状态展示

```
完成模式：会签
├── 负责人A ✅ 已完成（2026-04-15 10:30）
├── 负责人B ⏳ 未完成
└── 负责人C ✅ 已完成（2026-04-16 14:20）

进度：2/3 已完成
```

### 8.2 会签任务的完成逻辑

```
用户点击"完成" → 标记自己已完成
    ↓
POST /open-apis/task/v2/tasks/{task_guid}/complete
    ↓
自己的 completed_at 被记录
检查所有 assignee 是否都已完成：
├── 是 → 任务状态变为 done
└── 否 → 任务状态仍是 todo，显示 "X/Y 已完成"
```

### 8.3 获取会签任务完成信息

```
GET /open-apis/task/v2/tasks/{task_guid}
返回中包含：
{
  "assignee_related": [
    { "id": "ou_xxx", "completed_at": "1713000000000" },
    { "id": "ou_yyy", "completed_at": "0" }
  ]
}
```

- `completed_at` 为 "0" 表示该 assignee 尚未完成
- 有具体值表示已完成时间
