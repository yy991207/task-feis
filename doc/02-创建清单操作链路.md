# 飞书"创建任务清单 → 空白清单"操作链路

> 从用户操作视角拆解清单的完整生命周期
> **MVP 范围：仅任务中心创建任务，其他数据源后续扩展**

---

## 一、创建清单（MVP 第一步）

**用户操作：** 点击"创建清单" → 输入名称 → 确认

**API：**
```
POST /open-apis/task/v2/tasklists
Body: {
  "name": "清单名称",        // 必填，最长100字符
  "members": [...],         // 可选，创建时就加成员
  "archive_tasklist": false // 可选，默认不归档
}
```

**创建完成后的初始状态：**

```
┌─────────────────────────────────────┐
│  清单名称: "XXX"                      │
│  GUID: 自动生成 (UUID)                │
│  创建者/所有者: 当前用户               │
│  成员列表: [只有创建者自己]            │
│  分组(sections): 有1个默认分组         │
│  任务数量: 0                          │
│  归档状态: 未归档                      │
│  URL: 自动生成 applink                │
└─────────────────────────────────────┘
```

**关键点：**
- 创建者自动成为清单的 `owner`
- 清单自动带有 **1 个默认分组**（section）
- 此时清单是"空白"的——有结构但没有任务

---

## 二、空白清单的可操作项（7 类）

### 操作 1：管理清单基本信息

| 操作 | 说明 | 权限要求 |
|------|------|----------|
| 修改清单名称 | 重命名，最长100字符 | owner 或 editor |
| 转移所有者 | 把清单交给别人，原 owner 可降为 editor/viewer/none | 仅 owner |
| 归档清单 | 归档后清单只读 | owner 或 editor |
| 删除清单 | 删除清单，清单内任务不删除（变成游离任务） | 仅 owner |

**转移所有者的特殊逻辑：**
```
PATCH /open-apis/task/v2/tasklists/{guid}
Body: {
  "tasklist": {
    "owner": { "id": "新owner的open_id", "type": "user" }
  },
  "update_fields": ["owner"],
  "origin_owner_to_role": "editor"  // 原owner变成什么角色：editor/viewer/none
}
```

**可更新字段（update_fields）：**
- `name` — 清单名称
- `owner` — 清单所有者
- `archive_tasklist` — 归档状态

---

### 操作 2：管理清单成员

| 操作 | 说明 |
|------|------|
| 添加成员 | 指定用户 + 角色（editor 可编辑 / viewer 只读） |
| 移除成员 | 把用户从清单中踢出 |
| 修改角色 | editor ↔ viewer 切换 |

**成员类型支持：** user（普通用户）

**角色说明：**
- `editor` — 可编辑清单和清单内任务
- `viewer` — 只读，只能查看

---

### 操作 3：管理分组（Sections）

**默认分组：** 创建清单时自动有 1 个默认分组

**用户可操作：**
- 重命名分组
- 新增分组
- 删除分组（分组内任务不会删，会移到其他分组）
- 调整分组顺序

**分组与任务的关系：**
- 任务创建时必须落入某个分组（通过 `section_guid` 指定）
- 如果不指定分组，自动落入默认分组
- 清单内任务列表按分组组织展示

---

### 操作 4：在清单中创建任务

**MVP 方式：任务中心直接新建任务（指定清单+分组）**
```
POST /open-apis/task/v2/tasks
Body: {
  "summary": "任务标题",
  "tasklists": [{
    "tasklist_guid": "清单GUID",
    "section_guid": "分组GUID"
  }],
  "members": [
    { "id": "xxx", "role": "assignee" }
  ],
  ...其他可选字段
}
```

> 注：MVP 不实现"把已有任务移入清单"的功能，后续扩展时再做。

---

### 操作 5：管理自定义字段

自定义字段是**清单级别**的配置。

**流程：**
```
1. 清单 owner/editor 在清单设置中定义自定义字段
      ├── 字段名称（如"优先级""模块""状态"）
      ├── 字段类型（number/member/datetime/single_select/multi_select/text）
      └── 选项（如果是单选/多选类型，需预设选项列表）

2. 创建/编辑清单内任务时，这些字段出现在任务表单中

3. 填写字段值后，任务详情中展示这些值
```

**自定义字段类型：**

| 类型 | 值字段 | 说明 |
|------|--------|------|
| number | `number_value` | 数字，如 "10.23" |
| member | `member_value` | 人员数组，[{id, type, name}] |
| datetime | `datetime_value` | 时间戳字符串，如 "1698192000000" |
| single_select | `single_select_value` | 选项 GUID |
| multi_select | `multi_select_value` | 选项 GUID 数组 |
| text | `text_value` | 文本，最长3000字符 |

---

### 操作 6：查看清单任务列表

```
GET /open-apis/task/v2/tasklists/{guid}/tasks
Query:
  completed=true/false   # 按完成状态过滤
  created_from=时间戳     # 创建时间范围
  created_to=时间戳
  page_size=N            # 分页
  page_token=xxx
```

**返回结构：**
```json
{
  "has_more": false,
  "items": [
    {
      "guid": "任务GUID",
      "summary": "任务标题",
      "status": "todo/done",
      "section_guid": "分组GUID",
      ...
    }
  ],
  "page_token": ""
}
```

### 操作 6.1：字段配置（列表视图工具栏）

实际产品中，清单任务列表视图的工具栏有"字段配置"入口：
- 可选择在列表中显示/隐藏哪些列（负责人、截止时间、优先级、自定义字段等）
- 列顺序可拖拽调整
- 配置按清单独立保存

> 注：MVP 使用固定列展示（标题、负责人、截止时间、优先级），字段配置功能后续迭代。

---

### 操作 7：分享清单

- 每个清单有自动生成的 `url`（applink）
- 分享给其他人后，对方点击链接：
  - 如果已有权限 → 直接打开清单
  - 如果无权限 → 提示无权限或申请加入

## 三、完整操作流程图

```
用户点击"创建清单"
    │
    ├── 输入名称 → 确认
    │       ↓
    ├── 得到空白清单
    │       │
    │       ├── ① 重命名清单
    │       │
    │       ├── ② 添加成员
    │       │       ├── 加为 editor（可协作编辑）
    │       │       └── 加为 viewer（只读）
    │       │
    │       ├── ③ 管理分组
    │       │       ├── 重命名默认分组
    │       │       ├── 新增分组（如"待办""进行中""已完成"）
    │       │       └── 删除/排序分组
    │       │
    │       ├── ④ 配置自定义字段（可选）
    │       │       ├── 添加字段（优先级/模块/类型等）
    │       │       └── 设置字段选项
    │       │
    │       ├── ⑤ 在清单中创建任务
    │       │       ├── 选择目标分组
    │       │       ├── 填写任务信息
    │       │       ├── 指定负责人
    │       │       └── 设置时间/提醒/自定义字段
    │       │
    │       ├── ⑥ 转移所有权（交给别人管理）
    │       │
    │       └── ⑦ 归档/删除清单
    │               ├── 归档 → 只读，保留数据
    │               └── 删除 → 清单消失，任务变成游离任务
    │
    └── 清单不再空白，进入正常使用状态
```

---

## 四、关键逻辑总结

| 逻辑点 | 规则 |
|--------|------|
| 创建清单时 | 自动带1个默认分组，创建者是owner |
| 空白的含义 | 有分组结构但没有任务 |
| 任务归属 | 必须属于某个清单的某个分组 |
| 自定义字段 | 清单级别配置，任务级别填写 |
| 删除清单 | 任务不会被删，只是失去清单归属 |
| 归档清单 | 清单变只读，不能再创建/编辑任务 |
| 成员权限 | owner > editor > viewer，逐级递减 |
| 任务跨清单 | `tasklists` 是数组，实际产品支持一个任务属于多个清单。**MVP 暂不实现，一个任务只归属一个清单** |
