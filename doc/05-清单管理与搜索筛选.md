# MVP 范围：清单管理、搜索与筛选

> 清单的创建/编辑/成员管理，以及任务的搜索筛选
> 2026-04-17

## 一、清单管理完整逻辑

### 1.1 创建清单

```
任务中心 → "清单" tab → 点击"创建清单"
    ↓
输入清单名称（必填，最长100字符）
    ↓
POST /open-apis/task/v2/tasklists
Body: { "name": "清单名称" }
    ↓
返回：
{
  "tasklist": {
    "guid": "a08c3acf-...",
    "name": "清单名称",
    "owner": { "id": "ou_current_user", "type": "user" },
    "creator": { "id": "ou_current_user", "type": "user" },
    "members": [],
    "archive_msec": "0",
    "created_at": "1776000000000",
    "updated_at": "1776000000000",
    "url": "https://applink.feishu.cn/client/todo/tasklist/..."
  }
}
    ↓
清单创建成功
自动带有1个默认分组
创建者成为 owner
```

### 1.2 获取清单列表

```
任务中心 → "清单" tab
    ↓
GET /open-apis/task/v2/tasklists
    ↓
返回用户有权限访问的所有清单
```

### 1.3 获取清单详情

```
点击某个清单
    ↓
GET /open-apis/task/v2/tasklists/{tasklist_guid}
    ↓
返回清单信息（名称、owner、成员列表）
同时加载清单内任务列表：
GET /open-apis/task/v2/tasklists/{tasklist_guid}/tasks
```

### 1.4 编辑清单

**可编辑的内容：**

| 操作 | 字段 | 权限 |
|------|------|------|
| 重命名 | `name` | owner 或 editor |
| 转移所有者 | `owner` | 仅 owner |
| 归档 | `archive_tasklist` | owner 或 editor |

**API：**
```
PATCH /open-apis/task/v2/tasklists/{tasklist_guid}
Body: {
  "tasklist": {
    "name": "新名称"
  },
  "update_fields": ["name"]
}
```

### 1.5 管理清单成员

**添加成员：**
```
清单页面 → 设置 → 添加成员 → 选择用户 + 角色
    ↓
POST /open-apis/task/v2/tasklists/{tasklist_guid}/members
Body: {
  "members": [
    { "id": "ou_xxx", "role": "editor" },
    { "id": "ou_yyy", "role": "viewer" }
  ]
}
```

**移除成员：**
```
清单页面 → 设置 → 移除成员
    ↓
POST /open-apis/task/v2/tasklists/{tasklist_guid}/members/remove
Body: {
  "members": [
    { "id": "ou_xxx", "role": "editor" }
  ]
}
```

### 1.6 归档清单

```
清单页面 → 设置 → 归档
    ↓
PATCH /open-apis/task/v2/tasklists/{tasklist_guid}
Body: {
  "tasklist": { "archive_tasklist": true },
  "update_fields": ["archive_tasklist"]
}
    ↓
清单变为只读
不能再创建/编辑任务
清单内任务仍可查看
```

### 1.7 删除清单

```
清单页面 → 设置 → 删除
    ↓
DELETE /open-apis/task/v2/tasklists/{tasklist_guid}
    ↓
清单删除
清单内任务变成游离任务（不删除）
```

## 二、清单分组管理（清单的分组，非任务分组）

### 2.0 清单本身的分组

实际产品中，左侧导航栏的清单可以被分组管理（不同于清单内的任务分组 Section）：

```
左侧清单导航
├── 测试分组（清单分组）
│   ├── 任务清单1
│   └── 任务清单2
├── 项目分组
│   └── 4月迭代清单
└── 未分组
    └── 临时清单
```

- 用户可以创建"清单分组"来组织多个清单
- 清单可以拖拽到不同分组中
- 清单分组可重命名、删除
- 删除清单分组时，内部清单不删除，回到"未分组"

> 注：这是清单管理层面的分组，与清单内部的 Section（任务分组）是两个不同概念。MVP 实现基础的清单分组展示和管理。

## 三、分组(Section)管理

### 3.1 分组的特性

- 创建清单时自动生成 1 个默认分组
- 分组属于清单，不能跨清单
- 任务创建时必须落入某个分组
- 删除分组时，分组内任务不删除，移到其他分组

### 3.2 分组操作（MVP）

| 操作 | 说明 |
|------|------|
| 重命名分组 | 给分组起名字，如"待办""进行中""已完成" |
| 新增分组 | 添加新的分组 |
| 删除分组 | 删除前需选择目标分组迁移任务 |
| 排序分组 | 拖拽调整分组顺序 |

> 注：分组的 API 在 `tasklists` 返回中体现为分组信息，具体创建/编辑分组的 API 需通过清单详情或前端交互确认。

### 3.3 分组与任务的关系

```
清单
├── 分组A（section_guid: xxx）
│   ├── 任务1（tasklists[0].section_guid = xxx）
│   └── 任务2
├── 分组B（section_guid: yyy）
│   └── 任务3
└── 分组C（section_guid: zzz）
    └── 任务4
```

**任务移动到不同分组：**
```
PATCH /open-apis/task/v2/tasks/{task_guid}
Body: {
  "task": {
    "tasklists": [{
      "tasklist_guid": "清单GUID",
      "section_guid": "新分组GUID"
    }]
  },
  "update_fields": ["tasklists"]
}
```

## 四、任务搜索与筛选

### 4.1 搜索入口

```
任务中心顶部 → 搜索框
    ↓
输入关键词 → 实时搜索 / 回车搜索
    ↓
POST /open-apis/task/v2/tasks/search
Body: { "query": "关键词" }
```

### 4.2 高级筛选

**筛选条件面板：**
```
筛选
├── 负责人：[选择人员]（assignee）
├── 关注人：[选择人员]（follower）
├── 创建者：[选择人员]（creator）
├── 完成状态：[全部 / 未完成 / 已完成]（completed）
└── 截止时间：[选择日期范围]（due）
```

**API 调用示例：**
```
POST /open-apis/task/v2/tasks/search
Body: {
  "query": "智能体",
  "assignee": "ou_xxx",
  "completed": false,
  "due": "2026-04-01,2026-04-30"
}
```

### 4.3 时间范围格式

| 格式 | 示例 | 说明 |
|------|------|------|
| 日期 | `date:2026-04-01,date:2026-04-30` | 最常用 |
| ISO 8601 | `2026-04-01T00:00:00Z,2026-04-30T23:59:59Z` | 精确到秒 |
| 相对时间 | `relative:-7d,relative:0d` | 7天前到今天 |
| 时间戳 | `1712000000000,1714600000000` | 毫秒级 |

### 4.4 搜索结果展示

```
搜索结果页
├── 共找到 X 个任务
├── 任务列表（卡片或列表视图）
│   ├── [ ] 任务1标题 - 负责人: A - 截止: 4/20
│   ├── [ ] 任务2标题 - 负责人: B - 截止: 4/25
│   └── ...
└── 分页（如果结果多）
```

## 五、清单任务列表

### 5.1 查看清单内任务

```
进入清单页面
    ↓
GET /open-apis/task/v2/tasklists/{tasklist_guid}/tasks
Query:
  completed=false          # 只看未完成
  created_from=1712000000  # 创建时间范围
  page_size=50             # 每页条数
  page_token=xxx           # 分页
    ↓
返回按分组组织的任务列表
```

### 5.2 返回结构

```json
{
  "has_more": false,
  "items": [
    {
      "guid": "任务GUID",
      "summary": "任务标题",

      "created_at": "1776000000000",
      "tasklists": [{
        "tasklist_guid": "清单GUID",
        "section_guid": "分组GUID"

    }

  "page_token": ""

```

```
清单名称：4月迭代

│   ├── [ ] 任务1
│   ├── [ ] 任务2
```

## 七、关键边界情况

| 场景 | 处理方式 |
|------|----------|
| 清单名称为空 | 禁止创建，提示"名称不能为空" |
| 清单名称超长 | 限制100字符 |
| 添加已存在的成员 | 忽略或提示"已在清单中" |
| 最后一个 editor 被移除 | 清单只剩 owner，允许 |
| 删除最后一个分组 | 禁止，至少保留1个分组 |
| 归档后创建任务 | 禁止，提示"清单已归档" |
| 非清单成员创建任务 | 禁止，提示"无权限" |
| 搜索结果无结果 | 显示空状态"没有找到匹配的任务" |
| 清单无任务 | 显示空状态引导创建任务 |
